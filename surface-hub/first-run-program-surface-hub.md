# <a name="first-run-program-surface-hub"></a>Программа первого запуска (Surface Hub)


Термин "первый запуск" означает последовательность действий, выполняемых после первого включения Microsoft Surface Hub. Это синоним термина "запуск при первом включении компьютера". Этот процесс подробно описывается в данном разделе.

К текущему моменту вы уже должны были выполнить все предыдущие действия:

-   [подготовка среды для использования Surface Hub;](prepare-your-environment-for-surface-hub.md)
-   [физическая установка устройства Surface Hub](physically-install-your-surface-hub-device.md);
-   [подготовка рабочего плана настройки.](setup-worksheet-surface-hub.md)

Если это так, то первый запуск пройдет легко и быстро.
Обычная процедура состоит из шести шагов:

1.  [страница приветствия;](#first-page)
2.  [страница настройки;](#set-up-for-you)
3.  [страница учетной записи устройства;](#device-account)
4.  [страница ввода имени устройства;](#name-this-device)
5.  [страница настройки администраторов устройства;](#setup-admins)
6.  [обновление Surface Hub.](#update-surface-hub)

Каждый из этих разделов также содержит сведения о различных вариантах процедуры. Например, большинство устройств Surface Hub используют подключение к проводной сети, а некоторые— к беспроводной. Где это необходимо, приведены соответствующие сведения.

>[!NOTE]
>Перед началом вам следует настроить и подготовить отдельную клавиатуру, которая поставляется вместе с Surface Hub. Подробнее см. в руководстве по настройке Surface Hub.

 

## <a name="hi-there-page"></a><a href="" id="first-page"></a>Страница приветствия


Это первый экран, который вы увидите после первого включения Surface Hub. Здесь вы вводите информацию о локализации вашего устройства.

>[!NOTE]
>Кроме того, здесь можно развернуть пакет подготовки при необходимости. Если это так, изучите раздел [Создание пакетов подготовки](provisioning-packages-for-certificates-surface-hub.md).

 Выберите язык и параметры начальной настройки для отображения.

![Изображение, на котором показан контрольный список параметров ICD.](images/setuplocale.png)

### <a name="details"></a>Подробности

Если отображаются правильные значения по умолчанию, нажмите кнопку **Далее** для продолжения. В противном случае вам придется ввести данные в соответствующих полях.

-   **Страна или регион:** выберите страну или регион, где будет использоваться Surface Hub.
-   **Язык приложения:** приложения и компоненты будут использовать этот язык и формат языка.
-   **Раскладка клавиатуры:** выберите раскладку экранной и физической клавиатуры, которые будут использоваться с устройством.
-   **Часовой пояс:** выберите часовой пояс, в котором будет использоваться Surface Hub.

### <a name="what-happens"></a>Описание процедуры

>[!NOTE]
> После ввода параметров на этой странице вы сможете вернуться на этот экран, только сбросив устройство (см. [Сброс устройства](device-reset-surface-hub.md)). Перед продолжением убедитесь, что параметры настроены правильно.

 

Когда параметры будут приняты, устройство проверит наличие подключения к проводной сети. Если подключение установлено, появится [страница настройки](#set-up-for-you). Если с проводным подключением возникли проблемы, откроется [страница настройки сети](#network-setup).

Если проводное подключение отсутствует, устройство попытается настроить беспроводное соединение, и откроется страница [страница настройки сети](#network-setup).

## <a name="network-setup-page"></a><a href="" id="network-setup"></a>страница настройки сети


Вы увидите эту страницу, если устройство не найдет проводное подключение, которое можно использовать для подключения к сети или к Интернету. Здесь можно подключиться к беспроводной сети или вовсе пропустить этап подключения.

![Изображение, на котором показана страница настройки сети.](images/setupnetworksetup-1.png)

### <a name="details"></a>Подробности

Этот экран отображается, только если устройству не удается обнаружить проводную сеть. Если вы видите этот экран, у вас есть три варианта.

-   Можно выбрать одну из показанных беспроводных сетей. Если сеть защищена, вы будете перенаправлены на страницу входа. См. раздел [Настройка беспроводной сети](#wireless).
-   Щелкните **Пропустить этот шаг**, чтобы пропустить подключение к сети. Вы попадете на страницу [настройки](#set-up-for-you).
    >[!NOTE]
    >Если пропустить этот шаг, устройство не будет подключено к сети, и все функции, для которых требуется сетевое подключение, не будут работать на устройстве Surface Hub, в том числе обновление системы и синхронизация почты и календаря. Вы можете подключиться к беспроводной сети позже с помощью Параметры (см. [управление беспроводной сетью).](wireless-network-management-for-surface-hub.md)

     

-   Вы можете подключить сетевой кабель, пока отображается этот экран. Устройство обнаружит его, а на экране появится кнопка **Далее** . Нажмите **Далее** , чтобы продолжить подключение к проводной сети.

### <a name="what-happens"></a>Описание процедуры

Если устройство нашло проводное подключение при запуске и может подключиться к сети или к Интернету, то эта страница не отображается. Если вы хотите подключить устройство к беспроводной сети, убедитесь, что при первом запуске кабель Ethernet не подключен, чтобы открылся этот экран. Независимо от того, какие настройки вы выбрали, вы можете [использовать приложение "Параметры"](wireless-network-management-for-surface-hub.md) , чтобы настроить другие подключения позднее.

Чтобы подключиться к защищенной беспроводной сети на этой странице, щелкните нужную сеть и введите необходимые данные (пароль или учетные данные). См. раздел [Настройка беспроводной сети](#wireless).

## <a name="device-account-page"></a><a href="" id="device-account"></a>страница учетной записи устройства;


На этой странице Surface Hub запрашивает учетные данные для ранее настроенной учетной записи устройства. (См. раздел [Создание и тестирование учетной записи устройства](create-and-test-a-device-account-surface-hub.md).) Surface Hub попытается обнаружить различные свойства учетной записи и может запросить дополнительные сведения на другой странице.

>[!NOTE]
>В этом разделе не описываются конкретные ошибки, которые могут возникнуть во время первого запуска. Более подробную информацию об ошибках см. в разделе [Устранение неполадок Surface Hub](troubleshoot-surface-hub.md).


![Изображение, на котором показана страница ввода сведений для учетной записи устройства.](images/setupdeviceacct.png)

### <a name="details"></a>Подробности

В качестве идентификатора учетной записи введите в первом поле **имя участника-пользователя (UPN)** или **домен\\имя пользователя**. Используйте формат, соответствующий вашей среде, и введите пароль.


|                      Среда                      | Требуемый формат учетной записи устройства |
|-------------------------------------------------------|------------------------------------|
|         Учетная запись устройства размещается только в сети.         |        имя_пользователя@domain.com         |
|        Учетная запись устройства размещается только в локальной среде.         |          ДОМЕН\имя_пользователя           |
| Учетная запись устройства размещается в сети и локально (гибридный режим). |          ДОМЕН\имя_пользователя           |

Щелкните **Пропустить настройку учетной записи устройства**, чтобы пропустить этап настройки учетной записи. Однако если вы не настроите учетную запись, устройство не будет полностью интегрировано в вашу инфраструктуру. Например, пользователи не смогут:

-   видеть календарь собраний на экране приветствия;
-   начинать собрание на экране приветствия;
-   отправлять по электронной почте доски из OneNote;
-   использовать Skype для бизнеса для собраний.

Если пропустить настройку учетной записи, вы сможете добавить ее позже с помощью приложения "Параметры".

Если щелкнуть **Пропустить настройку учетной записи устройства**, появится диалоговое окно с описанием того, что произойдет, если на устройстве нет учетной записи. Если выбрать вариант **Да, пропустить**, откроется [страница ввода имени устройства](#name-this-device).

![Изображение, отображающее сообщение, отображающееся, чтобы подтвердить, что необходимо пропустить создание учетной записи устройства.](images/setupskipdeviceacct.png)

### <a name="what-happens"></a>Описание процедуры

Устройство будет использовать имя участника-пользователя или имя и сочетание "домен\\имя пользователя" и пароль учетной записи устройства для выполнения следующих задач.

-   Проверка наличия учетной записи в Active Directory (AD) или Azure Active Directory.

    -   Если введено имя участника-пользователя, устройство попробует найти учетную запись в Azure AD.
    -   Если введен идентификатор в формате "домен\\имя пользователя", устройство попробует найти учетную запись в AD.
-   Поиск сервера Microsoft Exchange Server для почтового ящика учетной записи.
-   Поиск SIP-адреса для учетной записи.
-   Получение отображаемого имени и псевдонима учетной записи.

## <a name="exchange-server-page"></a><a href="" id="exchange-server"></a>Страница Exchange Server


Эта страница отображается только при возникновении проблемы. Как правило, это означает, что указанная учетная запись устройства обнаружена в Active Directory (AD) или Azure Active Directory (Azure AD), но сервер Exchange Server для нее не был найден.

![Изображение, на котором показана страница сервера Exchange Server.](images/setupexchangeserver-01.png)

### <a name="details"></a>Подробности

Введите имя сервера Exchange Server, где размещается почтовый ящик учетной записи устройства.

Щелкните **Пропустить настройку служб Exchange** , чтобы пропустить этот шаг. В этом случае пользователи не смогут:

-   видеть календарь собраний на экране приветствия;
-   начинать собрание на экране приветствия;
-   отправлять по электронной почте доски из OneNote.

Более подробную информацию о зависимостях установки см. в разделе [Введение в Surface Hub](intro-to-surface-hub.md).

Вы можете включить службы Exchange для учетной записи устройства позднее в приложении "Параметры".

Если щелкнуть **Пропустить настройку служб Exchange**, появится диалоговое окно с описанием того, что произойдет. Если выбрать вариант **Да, пропустить**, службы Exchange не будут настроены.

![Изображение, на котором показано сообщение подтверждения, появляющееся при пропуске настройки служб Exchange.](images/setupexchangeserver-02.png)

### <a name="what-happens"></a>Описание процедуры

Surface Hub попытается проверить учетную запись устройства на сервере Exchange Server, который вы указали здесь. Если сервер Exchange Server доступен, процедура первого запуска продолжится.

Если пропустить настройку служб Exchange, Surface Hub перестанет искать сервер Exchange Server, а все службы Exchange (почта и календарь) будут отключены.

## <a name="exchange-policies-page"></a><a href="" id="exchange-policies"></a>Страница политик Exchange


Эта страница отобразится, если:

-   учетная запись устройства использует политику Exchange Active Sync (EAS), где политика PasswordEnabled имеет значение 1;
-   нет подключения к Exchange;
-   Exchange возвращает код ошибки (например, учетная запись подготовлена для слишком большого числа устройств);
-   поддерживаемые протоколы Exchange не поддерживаются Surface Hub;
-   Exchange возвращает недопустимый XML-код.

![Изображение, на котором показана страница политик Exchange.](images/setupexchangepolicies.png)

### <a name="details"></a>Подробности

Это информационная страница, поэтому ничего вводить не требуется. Однако у вас есть два варианта: продолжить или повторить проверку, которая вызвала ошибку. Прежде чем определить, какой вариант лучше, изучите следующий раздел **Описание процедуры** . Возможно, вы сможете устранить проблему, прежде чем выберите один из вариантов.

-   **Щелкните здесь, чтобы продолжить использование неподдерживаемых политик**: щелкните, чтобы продолжить процедуру первого запуска. Surface Hub не сможет использовать службы Exchange Server или синхронизацию.
-   **Повтор**: еще раз проверить политику на сервере Exchange Server.

### <a name="what-happens"></a>Описание процедуры

Surface Hub проверяет, задано ли для политики PasswordEnabled EAS учетной записи устройства значение 0 (False). Если это не так, почта и календарь не будут синхронизироваться, а Surface Hub не сможет использовать службы Exchange. Вы можете воспользоваться инструментами управления Exchange на компьютере, чтобы убедиться, что для политики PasswordEnabled задано значение 0. Если это не так, вы можете повторно настроить учетную запись и нажать кнопку **Повтор** на этой странице.

Если политика уже настроена правильно, убедитесь, что устройство подключено к сети или к Интернету и может связаться с сервером Exchange Server, поскольку эта страница также отображается, если Surface Hub не может подключиться к серверу Exchange Server.

Другая возможная причина отсутствия связи с Exchange — проверка подлинности на основе сертификатов. Вы можете оказаться на этой странице из-за проблем с сертификатом. Обратите внимание, что если отображается код ошибки 0x80072F0D или 0X800C0019, требуется сертификат. Поскольку подготовка выполняется на первой странице процесса первого запуска, следует отключить службы Exchange, нажав кнопку **Щелкните здесь, чтобы продолжить использование неподдерживаемых политик**, а затем установить соответствующие сертификаты с помощью приложения "Параметры".

Если пропустить эту проверку, Surface Hub перестанет искать сервер Exchange Server и проверять политики EAS, а службы Exchange Server будут отключены. Более подробную информацию о зависимостях установки см. в разделе [Введение в Surface Hub](intro-to-surface-hub.md) .

## <a name="name-this-device-page"></a><a href="" id="name-this-device"></a>страница ввода имени устройства;


На этой странице вам будет предложено ввести два имени, которые используются для идентификации Surface Hub.

![Изображение, на котором показана страница ввода имени устройства.](images/setupnamedevice.png)

### <a name="details"></a>Подробности

Если отображаются правильные значения по умолчанию, нажмите кнопку **Далее** для продолжения. В противном случае введите данные в одном или обоих текстовых полях.

-   **Понятное имя:** это имя пользователи будут видеть при беспроводном подключении к Surface Hub.
-   **Имя устройства:** здесь можно ввести любое уникальное имя, как указано на экране.

Если оба имени соответствуют требованиям к длине и не содержат недопустимые символы, нажмите кнопку **Далее** , чтобы перейти на следующую страницу, [Настроить администраторов для этого устройства](#setup-admins).

### <a name="what-happens"></a>Описание процедуры

Surface Hub запрашивает два имени для устройства. По умолчанию используются следующие данные.

-   **Понятное имя:** отображаемое имя учетной записи устройства по умолчанию.
-   **Имя устройства:** по умолчанию это псевдоним учетной записи устройства.

Оба имени можно изменить позже, но помните:

-   понятное имя должно быть узнаваемым и различимым, чтобы пользователи могли отличить разные устройства Surface Hub при беспроводном подключении к ним;
-   если вы решите присоединить устройство к домену, имя устройства не должно совпадать с именем любого другого устройства в домене Active Directory учетной записи; устройство не может быть присоединено к домену, если его имя совпадает с именем другого устройства, присоединенному к домену.

>[!NOTE]
>Если вы хотите включить [Miracast над инфраструктурой](miracast-over-infrastructure.md), должно быть возможно обнаружение имени устройства через DNS. Этого можно добиться либо разрешив Surface Hub автоматически регистрироваться через динамический DNS, либо путем создания записи A или AAAA для имени устройства Surface Hub вручную.

## <a name="set-up-admins-for-this-device-page"></a><a href="" id="setup-admins"></a>страница настройки администраторов устройства;


На этой странице вы выберете один из вариантов настройки учетных записей администраторов для локального управления устройством.

Поскольку каждое устройство Surface Hub можно использовать с любым количеством сотрудников, прошедших проверку подлинности, параметры заблокированы, чтобы они не менялись в разных сеансах. Только администраторы могут настроить параметры на устройстве. На этой странице вы выберете, какой тип администраторов сможет это делать.

>[!NOTE]
>Примечание. Основная цель этой страницы— определить, кто сможет настраивать устройство в пользовательском интерфейсе, то есть входить в систему на устройстве, открывать приложение "Параметры" и изменять настройки.

 

![Изображение, на котором показана страница настройки администраторов для этого устройства.](images/setupsetupadmins.png)

### <a name="details"></a>Подробности

Выберите один из трех доступных вариантов:

-   **использовать Microsoft Azure Active Directory;**
-   **использовать доменные службы Active Directory;**
-   **использовать локального администратора.**

### <a name="what-happens"></a>Описание процедуры

Вот, что происходит при выборе параметра.

-   **использовать Microsoft Azure Active Directory;**

    Если выбрать этот параметр, вы сможете присоединить устройство к Azure AD. После нажатия кнопки **Далее**устройство перезагружается для применения некоторых параметров, а затем открывается страница [Использовать Microsoft Azure Active Directory](#use-microsoft-azure) , где вам будет предложено ввести учетные данные для присоединения к Azure AD. Члены роли глобальных администраторов Azure из присоединяемой организации смогут использовать приложение Параметры. Пользователи, которым это разрешено, зависят от подписки Azure AD и параметров организации Azure AD.
    
    > [!IMPORTANT]
    > Чтобы настроить, кто может использовать приложение Параметры для администрирования концентраторов Surface, убедитесь, что автоматическая регистрация [intune](/mem/intune/enrollment/windows-enroll#enable-windows-10-automatic-enrollment) включена в клиенте перед присоединением устройства к Azure AD. Затем политики intune можно использовать для настройки не глобальных администраторов [в](surface-hub-2s-nonglobal-admin.md) Surface Hubs.

-   **Использовать доменных служб Active Directory**

    Выберите этот параметр, чтобы присоединить устройство к AD. После нажатия кнопки **Далее**вы попадете на страницу [Использовать доменные службы Active Directory](#use-active-directory) , где вам будет предложено ввести учетные данные для присоединения к домену. После присоединения вы можете выбрать группу безопасности из присоединенного домена, а пользователи из этой группы смогут применять приложение "Параметры".

-   **использовать локального администратора.**

    Этот администратор не будет поддерживаться какой-либо службой каталогов, поэтому мы рекомендуем выбирать этот вариант, только если у устройства нет доступа к Azure AD или AD. После указания имени пользователя и пароля администратора на странице [Использовать локального администратора](#use-a-local-admin) вам потребуется вводить эти учетные данные при каждом открытии приложения "Параметры".

    Обратите внимание, что у локального администратора должен быть физический доступ к Surface Hub для входа в систему.

>[!NOTE]
>После завершения этого процесса вы сможете изменить параметры администратора, только сбросив устройство.

 

### <a name="use-microsoft-azure-active-directory"></a><a href="" id="use-microsoft-azure"></a>Использовать Microsoft Azure Active Directory

Если вы решили присоединить Surface Hub к службе Azure Active Directory (Azure AD), вы увидите эту страницу **Что произойдет далее**. Изучите ее и нажмите кнопку **Далее** , чтобы открыть **страницу входа**.

Присоединение Azure AD дает два основных преимущества:

1.  некоторые сотрудники вашей организации получат доступ к устройству в качестве администраторов и смогут запускать приложение "Параметры" и настроить устройство; пользователи с разрешениями администратора будут указаны в вашей подписке Azure AD.

2.  Если служба Azure AD подключена к решению управления мобильными устройствами (MDM), устройство будет зарегистрировано в нем, чтобы вы могли применить политики и конфигурацию.

    ![Изображение, на котором показано сообщение, появляющееся при присоединении Surface Hub к Azure Active Directory.](images/setupjoiningazuread-1.png)

### <a name="details"></a>Подробности

Требуется ввести следующие данные.

-   **Имя участника-пользователя:** это имя участника-пользователя (UPN) учетной записи, которая может присоединиться к Azure AD.
-   **Пароль:** пароль учетной записи, который вы используете для присоединения к Azure AD.

![Изображение, на котором показана страница ввода данных для входа.](images/setupjoiningazuread-2.png)

Если вы дошли до этой точки и у вас нет допустимых учетных данных для учетной записи Azure AD, устройство позволит создать локальную учетную запись администратора. Щелкните **Настроить Windows с помощью локальной учетной записи**.

![Изображение, на котором показана страница настройки учетной записи администратора.](images/setupjoiningazuread-3.png)

### <a name="what-happens"></a>Описание процедуры

После ввода допустимых учетных данных Azure AD устройство попытается присоединиться к соответствующей организации Azure AD. Если попытка будет успешной, устройство подготовит сотрудников организации, которые станут локальными администраторами. Если клиент Azure AD настроен для устройства, оно также будет зарегистрировано в MDM.

### <a name="use-active-directory-domain-services"></a><a href="" id="use-active-directory"></a>использовать доменные службы Active Directory;

На этой странице запрашиваются учетные данные для присоединения к домену, чтобы устройство Surface Hub могло подготовить участников группы безопасности как администраторов устройства.

После присоединения устройства к домену необходимо указать группу безопасности из домена, к которому вы присоединились. Пользователи этой группы безопасности будут подготовлены как администраторы Surface Hub, и все ее участники смогут ввести свои учетные данные домена для доступа к приложению "Параметры".

![Изображение, на котором показана страница настройки администраторов с использованием присоединения к домену.](images/setupdomainjoin.png)

### <a name="details"></a>Подробности

Требуется ввести следующие данные.

-   **Домен:** это полное доменное имя (FQDN) домена, к которому нужно присоединиться. Группу безопасности из этого домена можно использовать для управления устройством.
-   **Имя пользователя:** это имя пользователя учетной записи с необходимыми разрешениями для присоединения к указанному домену. 
-   **Пароль:** пароль для учетной записи.

После проверки учетных данных вам будет предложено ввести имя группы безопасности. Это поле является обязательным.

![Изображение, на котором показана страница ввода группы безопасности.](images/setupsecuritygroup-1.png)

### <a name="what-happens"></a>Описание процедуры

Используя указанный домен, учетные данные учетной записи со страницы [Использование доменных служб Active Directory](#use-active-directory) и имя устройства со страницы [ввода имени устройства](#name-this-device), Surface Hub попытается присоединиться к домену. Если попытка будет успешной, процедура первого запуска продолжится, а вам будет предложено ввести группу безопасности. Если присоединение завершилось ошибкой, процедура первого запуска будет приостановлена, а вам потребуется изменить введенные данные.

Если присоединение прошло успешно, откроется страница **ввода группы безопасности** . Если нажать кнопку **Выбрать** на этой странице, устройство выполнит поиск указанной группы безопасности в вашем домене. Если группа будет найдена, она будет подтверждена. Нажмите кнопку **Готово**, чтобы завершить процесс первого запуска.

>[!NOTE]
>Если присоединить Surface Hub к домену, вы не сможете отменить присоединение без сброса устройства.

 

### <a name="use-a-local-admin"></a>Использовать локального администратора

Если вы решили не использовать Azure Active Directory (Azure AD) или Active Directory (AD) для управления Surface Hub, вам нужно создать учетную запись локального администратора.

![Изображение, на котором показана страница настройки учетной записи для локального администратора.](images/setuplocaladmin.png)

### <a name="details"></a>Подробности

Требуется ввести следующие данные.

-   **Имя пользователя:** это имя пользователя учетной записи локального администратора, которая будет создана для этого устройства Surface Hub.
-   **Пароль:** это пароль учетной записи устройства.
-   **Введите пароль повторно:** введите тот же пароль, что и в предыдущем поле.

### <a name="what-happens"></a>Описание процедуры

На этой странице устройство попробует создать новую учетную запись администратора, используя введенные здесь учетные данные. Если попытка будет успешной, процедура первого запуска будет завершена. В противном случае вам понадобится ввести другие учетные данные.

## <a name="update-the-surface-hub"></a><a href="" id="update-surface-hub"></a>Обновление Surface Hub


>[!IMPORTANT]
>Перед обновлением прочитайте раздел [Сохранение ключа BitLocker](save-bitlocker-key-surface-hub.md), чтобы у вас была резервная копия ключа.

 

Чтобы использовать последние возможности и исправления, необходимо обновить Surface Hub сразу после завершения предыдущих этапов процедуры первого запуска.

1.  Убедитесь, что устройство имеет доступ к серверам Windows обновления. 
2.  Откройте приложение "Параметры", щелкните **Обновление и безопасность**, выберите **Центр обновления Windows**, а затем нажмите кнопку **Проверить наличие обновлений**.
3.  Если обновления доступны, они будут скачаны. После завершения скачивания нажмите кнопку **Обновить** для установки обновлений.
4.  Следуйте инструкциям на экране после установки обновлений. Может потребоваться перезапустить устройство.

# <a name="first-time-setup-for-surface-hub"></a>Первая настройка для Surface Hub

При первом запуске Surface Hub устройство автоматически переходит в режим первой установки, чтобы просмотреть конфигурацию учетной записи и соответствующие параметры.

> [!TIP]
> Вы можете автоматизировать весь процесс установки [с помощью пакета](#use-provisioning-packages) подготовки, чтобы обеспечить согласованный интерфейс на нескольких устройствах Surface Hub.

## <a name="get-started"></a>Начало работы

1. По умолчанию Кортана включено, чтобы выполнить этот процесс. Чтобы отключить Кортана помощь, щелкните значок микрофона.

    :::image type="content" source="images/hub-setup-cortana.png" alt-text="Кортана включено, чтобы выполнить этот процесс.":::

2. **Выберите регион.** Подтвердите автоматически обнаруженный регион и выберите " **Да"**.

    :::image type="content" source="images/hub-setup-region.png" alt-text="Выберите регион.":::

3. **Подтвердите раскладку клавиатуры.** Выберите **"Да"**.

    :::image type="content" source="images/hub-setup-keyboard.png" alt-text="Подтвердите раскладку клавиатуры.":::

4. Чтобы добавить вторую клавиатуру, нажмите **кнопку "Добавить макет"**. В противном случае выберите **"Пропустить"**.

    :::image type="content" source="images/hub-setup-2keyboard.png" alt-text="Добавьте вторую клавиатуру.":::

5. **Подключение в сеть.** Если вы уже подключили кабель Ethernet, Surface Hub автоматически подключается к сети. Кроме того, можно подключиться к беспроводной сети. **Примечание:** Вы не можете подключиться к беспроводной сети в хот-спотах (captive portals), которые перенаправляют запросы на вход на веб-сайт поставщика. Выберите **Далее**.

    :::image type="content" source="images/hub-setup-network.png" alt-text="Подключение в сеть.":::

6. **Примите Windows 10 лицензионного соглашения.** Выберите **"Принять"**.

    :::image type="content" source="images/hub-setup-license.png" alt-text="Примите Windows 10 лицензионного соглашения.":::

7. **Введите сведения об** учетной записи устройства с помощью имени участника-пользователя (user@contoso.com) или доменного адреса более высокого уровня (CONTOSO\user). Используйте формат, соответствующий вашей среде, и введите пароль.

    :::image type="content" source="images/hub-setup-device-account.png" alt-text="Введите сведения об учетной записи устройства.":::

| Среда                                              | Требуемый формат учетной записи устройства |
| -------------------------------------------------------- | ---------------------------------- |
| Учетная запись устройства размещена только в Интернете                     | username@contoso.com               |
| Учетная запись устройства размещена только в локальной среде                | CONTOSO\user                       |
| Учетная запись устройства размещена в сети и локально (гибридная) | CONTOSO\user                       |

>[!NOTE]
>Хотя вы можете пропустить настройку учетной записи устройства, устройство не будет полностью интегрировано в инфраструктуру. Если пропустить настройку учетной записи, вы сможете добавить ее позже с помощью приложения "Параметры".

8. **Введите пароль и** нажмите кнопку **"Далее".**

9. Surface Hub автоматически обнаруживает Exchange сервера и адрес SIP из домена, введенного на предыдущем шаге. При необходимости укажите адрес Exchange сервера и нажмите кнопку "**Далее"**.

    :::image type="content" source="images/hub-setup-exchange.png" alt-text="Exchange сервера и SIP-адреса.":::

10. **Приведите имя этого устройства.** Введите имя устройства или используйте предложенное. **Нажмите кнопку "Далее"**.

    :::image type="content" source="images/hub-setup-name.png" alt-text="Приведите имя этого устройства.":::

- **Понятное имя** отображается в левом нижнем углу Surface Hub 2S и отображается при проецирование на устройство.
- Имя **устройства определяет** устройство при связи с Active Directory или Azure Active Directory, а также при регистрации устройства в Intune.

>[!IMPORTANT]
>Если вы планируете прикрепить Surface Hub с Active Directory, имя устройства должно соответствовать стандартным требованиям к именам компьютеров в [AD](/troubleshoot/windows-server/identity/naming-conventions-for-computer-domain-site-ou#computer-names), в противном случае установка завершится ошибкой.

>[!NOTE]
>Если вы хотите включить [Miracast над инфраструктурой](miracast-over-infrastructure.md), должно быть возможно обнаружение имени устройства через DNS. Этого можно добиться либо разрешив Surface Hub автоматически регистрироваться через динамический DNS, либо путем создания записи A или AAAA для имени устройства Surface Hub вручную.

### <a name="configure-device-admin-accounts"></a>Настройка учетных записей администраторов устройств

Администраторы устройств можно настроить только во время первой установки. Дополнительные сведения см. в разделе:

- [Surface Hub устройства 2S](/surface-hub/prepare-your-environment-for-surface-hub#device-affiliation)
- [управление группой администраторов](admin-group-management-for-surface-hub.md)

1. **Выберите тип учетной записи администратора.** Выберите один из следующих вариантов: доменные службы Active Directory, Azure Active Directory или локальный администратор.

    :::image type="content" source="images/hub-setup-join.png" alt-text="Выберите тип учетной записи администратора.":::

### <a name="active-directory-domain-services"></a>Доменные службы Active Directory

1. Если вы планируете использовать Surface Hub в локальной среде, вы можете аффилировать Hub **с доменные службы Active Directory.**  Введите учетные данные пользователя, который имеет разрешения на присоединение устройства к Active Directory.
2. Выберите группу безопасности Active Directory, содержащую участников, которым разрешено входить в приложение Параметры на Surface Hub 2S.
3. Нажмите **кнопку "Готово"**. Устройство перезапустится.

### <a name="microsoft-azure-active-directory"></a>Microsoft Azure Active Directory

1. Если вы планируете управлять Surface Hub из облака с помощью Microsoft Intune или поставщика MDM, выберите **Microsoft Azure Active Directory**.
2. Нажмите кнопку "Далее" и войдите с помощью рабочей или учебной учетной записи. При перенаправлении выполните проверку подлинности на странице входа в свою организацию и при запросе укажите дополнительные учетные данные. В противном случае введите пароль и нажмите кнопку **"Далее".**

    :::image type="content" source="images/hub-setup-signin.png" alt-text="Войдите с помощью рабочей или учебной учетной записи.":::

>[!NOTE]
>Чтобы настроить, кто может использовать приложение Параметры для администрирования Surface Hubs, убедитесь, что автоматическая регистрация Intune включена в клиенте перед присоединением устройства к Azure AD. Intune политики можно использовать для настройки не глобальных администраторов [на](surface-hub-2s-nonglobal-admin.md) Устройствах Surface Hub.

### <a name="local-administrator-account"></a>Учетная запись локального администратора

- Введите имя пользователя и запоминающийся пароль для локального администратора. (Если вы забыли пароль локального администратора, вам потребуется [](surface-hub-2s-recover-reset.md) восстановить устройство и повторить процесс установки.)  

    :::image type="content" source="images/hub-setup-local-admin.png" alt-text="Введите запоминающийся пароль для учетной записи локального администратора.":::

### <a name="choose-privacy-settings-for-your-device"></a>Выбор параметров конфиденциальности для устройства

- Выберите из доступных параметров конфиденциальности и нажмите кнопку **"Принять".**

    :::image type="content" source="images/hub-setup-privacy.png" alt-text="Выберите параметры конфиденциальности.":::

### <a name="use-provisioning-packages"></a>Использование пакетов подготовки

Вы можете настроить параметры настройки при первом запуске, что позволит обеспечить согласованность взаимодействия между несколькими устройствами Surface Hub.

1. Для начала ознакомьтесь с документацией в разделе ["Создание](provisioning-packages-for-surface-hub.md) пакетов подготовки" и сохраните пакет подготовки на USB-накопителе.
2. Вставьте USB-накопитель в один из USB-портов, когда вы увидите страницу лицензионного соглашения (шаг 6 в шагах настройки выше).
3. При появлении запроса выберите пакет подготовки, который вы хотите использовать.
4. Если вы создали CSV-файл нескольких устройств, вы сможете выбрать конфигурацию устройства.
5. Следуйте инструкциям, чтобы завершить настройку при первом запуске.

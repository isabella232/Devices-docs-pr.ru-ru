# <a name="manage-windows-updates-on-surface-hub"></a>Обновления Windows на Surface Hub

Новые выпуски операционной Surface Hub публикуются через клиентский компонент Центра обновления Windows, как и выпуски Windows 10 или Windows 11. На этой странице приведены рекомендации по управлению обновлениями Surface Hub устройств. 

## <a name="windows-update-for-business"></a>Центр обновления Windows для бизнеса

клиентский компонент Центра обновления Windows для бизнеса — это набор функций, предназначенных для предоставления предприятиям дополнительного контроля над тем, как и клиентский компонент Центра обновления Windows устанавливать выпуски, одновременно уменьшая затраты на управление устройствами. С помощью этого метода устройства Surface Hub непосредственно подключаются к службе Microsoft "Центр обновления Windows".

- Получайте обновления непосредственно от службы Microsoft "Центр обновления Windows", дополнительная инфраструктура для этого не требуется. 
- Откладывайте обновления, чтобы получить дополнительное время для тестирования и оценки. 
- Развертывайте обновления на устройствах определенных групп. 
- Определите периоды обслуживания для установки обновлений. 

> [!TIP]
> Используйте одноранговый обмен содержимым, чтобы свести к минимуму проблемы с пропускной способностью во время обновлений. [Дополнительные сведения см. в Windows "Оптимизация доставки](/windows/deployment/do/waas-optimize-windows-10-updates) обновлений".

> [!NOTE]
> Surface Hub в настоящее время не поддерживает откат обновлений.


## <a name="surface-hub-servicing-model"></a>Модель обслуживания Surface Hub

Surface Hub использует модель Windows обслуживания, называемую Windows [как услуга (WaaS).](/windows/deployment/update/waas-overview) Традиционно новые возможности добавлялись только в новых версиях Windows, которые выпускались каждые несколько лет. Развертывание каждой новой версии в организации представляло собой длительный и дорогостоящий процесс. В результате конечные пользователи и сама организация нечасто пользовались преимуществами новых инноваций. Цель модели обслуживания "Windows как услуга" — постоянно предоставлять новые возможности, сохраняя высокий уровень качества.

Корпорация Майкрософт регулярно публикует два типа выпусков Surface Hub:
- **Обновления компонентов**— обновления, устанавливающие актуальные новые компоненты, элементы пользовательского интерфейса и возможности. Корпорация Майкрософт ожидает публикации двух новых обновлений компонентов в год.
- **Исправления**— обновления, целью которых является установка исправлений безопасности, драйверов и других служебных обновлений. Корпорация Майкрософт планирует публиковать одно накопительное исправление в месяц.

Чтобы улучшить качество выпуска и упростить развертывание, все новые выпуски, опубликованные корпорацией Майкрософт для Windows 10 или Windows 11, включая Surface Hub, будут накопительными. Это означает, что новые обновления компонентов и исправления включают в себя содержимое всех предыдущих выпусков (в оптимизированном формате, чтобы уменьшить требования к объему хранимых и передаваемых по сети данных), и после установки выпуска на устройство оно будет полностью обновлено. Кроме того, в отличие от предыдущих версий Windows, вы не сможете установить часть содержимого исправления для Windows 10. Например, если исправление содержит исправления для трех уязвимостей системы безопасности и одной проблемы надежности, при развертывании этого обновления будут установлены все четыре исправления.

Операционная система Surface Hub получает обновления по каналу [Semi-Annual Channel](/windows/deployment/update/waas-overview#naming-changes). Как и в других выпусках Windows 10 или Windows 11, время обслуживания ограничено. Вам необходимо установить обновления компонентов на компьютерах с этими типами обслуживания, чтобы продолжать получать ежемесячные исправления.

Дополнительные сведения о Windows как услуге см. в разделе [Обзор Windows как услуги](/windows/deployment/update/waas-overview).


## <a name="use-windows-update-for-business"></a>Использование Центра обновления Windows для бизнеса

Устройства Surface Hub, как все устройства Windows 10, включают **Центр обновления Windows для бизнеса (WUfB)**, что позволяет вам контролировать обновление ваших устройств. Эти возможности Центра обновления Windows для бизнеса позволяют снизить затраты на управление устройствами, наладить контроль над развертыванием обновлений, обеспечить более быструю доступность обновлений для системы безопасности и новейших инноваций Майкрософт на постоянной основе. Дополнительные сведения см. в разделе [Управление обновлениями с помощью Центра обновления Windows для бизнеса](/windows/deployment/update/waas-manage-updates-wufb).

> [!IMPORTANT]
> Как правило, корпорация Майкрософт выпускает одно обязательное обновление Windows безопасности в месяц (выпущено во второй вторник и часто называется выпуском B). Вместе с внештатными обновлениями безопасности это единственными обновлениями, доступными для устройств с помощью WUfB. Однако Surface Hub улучшения обычно предоставляются через необязательные обновления, не включаемые в безопасность, в третий вторник каждого месяца (выпуск C). В результате клиенты, использующие клиентский компонент Центра обновления Windows для бизнеса со своими Устройствами Surface Hub, будут ожидать выпуска B в следующем месяце, чтобы увидеть последние улучшения на этих устройствах.

**Настройка Центра обновления Windows для бизнеса**
1. [Объединение устройств Surface Hub в циклы развертывания](#group-surface-hub-into-deployment-rings)
2. [Настройка времени получения обновлений Surface Hub](#configure-when-surface-hub-receives-updates).

> [!NOTE]
> Для настройки WUfB Microsoft Intune, Microsoft Endpoint Configuration Manager или поддерживаемого стороннего поставщика MDM. [Пошаговое руководство: использование Microsoft Intune для настройки Центра обновления Windows для бизнеса.](/windows/deployment/update/waas-wufb-intune)


### <a name="group-surface-hub-into-deployment-rings"></a>Объединение устройств Surface Hub в циклы развертывания

Используйте циклы развертывания, чтобы контролировать время развертывания обновлений на устройствах Surface Hub, оставив достаточно времени для их проверки. Например, можно сначала обновить небольшой пул устройств, чтобы проверить качество обновления, и только после этого приступать к крупномасштабному развертыванию в организации. В зависимости от того, кто управляет Surface Hub в вашей организации, рассмотрите возможность включения Surface Hub в круги развертывания, созданные для других Windows 10 или Windows 11 устройств. Дополнительные сведения о кругах развертывания см. в статье "Подготовка стратегии обслуживания [Windows обновлений клиентов"](/windows/deployment/update/waas-servicing-strategy-windows-10-updates).

Примеры кругов развертывания см. в следующей таблице.

| Цикл развертывания | Размер цикла | Ветвь обслуживания | Отсрочка обновлений компонентов | Отсрочка исправлений (исправления безопасности, драйверы и другие обновления) | Шаг проверки |
| --------- | --------- | --------- | --------- | --------- | --------- |
| Оценка (например, некритический или проверка устройств) | Небольшой | Windows Insider Preview | Нет.  | Нет.  | Протестируйте и оцените новую функциональность вручную. Приостановите обновления, если возникают проблемы. |
| Пилотный проект (например, устройства, используемые избранными рабочими группами) | Средний | Semi-annual channel  | Нет. | Нет.  | Отслеживание использования устройств и отзывов пользователей. Приостановите обновления, если возникают проблемы. |
| Крупномасштабное развертывание (например, на большинстве устройств в вашей организации) | Крупный | Semi-annual channel |  120 дней после выпуска. | 7-14 дней после выпуска. | Отслеживание использования устройств и отзывов пользователей. Приостановите обновления, если возникают проблемы. |
| Критически важное (например, устройства в залах заседания исполнительного руководства) | Небольшой | Semi-annual channel |  180 дней после выпуска (максимальная отсрочка обновлений компонентов). | 30 дней после выпуска (максимальная отсрочка исправлений). | Отслеживание использования устройств и отзывов пользователей. |


### <a name="configure-when-surface-hub-receives-updates"></a>Настройка времени получения обновлений Surface Hub

Определив циклы развертывания для устройств Surface Hub, настройте политики отсрочки обновления для каждого цикла:
- Чтобы отложить обновления компонентов, настройте соответствующую политику [Update/DeferFeatureUpdatesPeriodInDays](/windows/client-management/mdm/policy-csp-update#update-deferfeatureupdatesperiodindays) для каждого цикла.
- Чтобы отложить исправления, настройте соответствующую политику [Update/DeferQualityUpdatesPeriodInDays](/windows/client-management/mdm/policy-csp-update#update-deferqualityupdatesperiodindays) для каждого цикла.

> [!NOTE]
> При возникновении проблем во время развертывания обновлений можно приостановить обновления с использованием политик [Update/PauseFeatureUpdates](/windows/client-management/mdm/policy-csp-update#update-pausefeatureupdates) и [Update/PauseQualityUpdates](/windows/client-management/mdm/policy-csp-update#update-pausequalityupdates).

**Если вы используете прокси-сервер или другой метод для блокировки URL-адресов**

Добавьте следующие Windows url-адреса надежных сайтов в список разрешений:
- `http(s)://*.update.microsoft.com`
- `http://download.windowsupdate.com` 
- `http://windowsupdate.microsoft.com`

После установки юбилейного обновления Windows 10 вы можете удалить эти адреса, чтобы вернуть Surface Hub в предыдущее состояние.

## <a name="maintenance-window"></a>Период обслуживания

Чтобы убедиться, что устройство всегда доступно для использования в рабочее время, Surface Hub выполняет свои административные функции во время заданных периодов обслуживания. Во время периода обслуживания Surface Hub автоматически устанавливает обновления через клиентский компонент Центра обновления Windows и перезагружает устройство за 20 минут до конца окна.

Surface Hub применяет обновления, следуя нижеперечисленным принципам:
- Установка обновления в следующий период обслуживания. Если собрание запланировано на период обслуживания или датчики Surface Hub обнаружат, что устройство используется, обновление будет отложено до следующего периода обслуживания.
- Если следующий период обслуживания наступит только после предписанной отсрочки для установки обновления, устройство вычислит следующий доступный период в рабочее время. Для этого будет использовано приблизительное время установки из метаданных обновления. Система продолжит откладывать обновление, если запланировано собрание или датчики Surface Hub обнаружат, что устройство используется.
- Если в следующем периоде обслуживания **** не истек льготный период обновления, Surface Hub продолжит откладывать обновление.
- Если требуется перезагрузка, Surface Hub автоматически выполнит ее в следующий период обслуживания.

> [!NOTE]
> При первой настройке Surface Hub необходимо отвести определенное время для обновлений. Например, может появиться журнал невыполненных работ по определениям вирусов, который следует установить немедленно.

Период обслуживания по умолчанию установлен для всех новых устройств Surface Hub.
-   **Время начала:** 02:00
-   **Длительность:** 2 часа

**Чтобы изменить период обслуживания вручную, выполните следующие действия:**
1.  Откройте **Параметры** на своем устройстве Surface Hub.
2.  Перейдите в раздел **Обновление и безопасность** > **Центр обновления Windows** > **Дополнительные параметры**.
3.  В разделе **Часы обслуживания**нажмите кнопку **Изменить**.

Чтобы изменить период обслуживания с помощью MDM, задайте узел **MaintenanceHoursSimple** в поставщике служб [конфигурации SurfaceHub](/windows/client-management/mdm/surfacehub-csp). См. дополнительные сведения в разделе [Управление параметрами с помощью поставщика MDM](manage-settings-with-mdm-for-surface-hub.md).

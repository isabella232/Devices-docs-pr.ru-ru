# Регистрация устройств Surface в SEMM и их настройка

В режиме управления Microsoft Surface Enterprise (SEMM) можно безопасно настроить параметры UEFI Surface на устройстве Surface и управлять этими настройками на устройствах Surface в организации. Если устройство Surface управляется SEMM, это устройство считается зарегистрированным *(иногда* называется активированным). В этой статье показано, как создать пакет конфигурации UEFI Surface, который будет не только управлять настройками UEFI Surface, но и зарегистрировать устройство Surface в SEMM.

Более общие сведения о SEMM см. в [режиме управления Microsoft Surface Enterprise.](https://technet.microsoft.com/itpro/surface/surface-enterprise-management-mode)

В качестве альтернативы SEMM новые устройства Surface поддерживают удаленное управление подмножество параметров микропрограмм с помощью Microsoft Intune. Дополнительные сведения см. в управлении [Intune для параметров UEFI Surface.](surface-manage-dfci-guide.md)

> [!NOTE]
> SeMM поддерживается на Surface Pro X только с помощью диспетчера UEFI. Дополнительные сведения о развертывании, управлении и обслуживании [Surface Pro X.](surface-pro-arm-app-management.md)

#### Скачивание и установка Microsoft Surface UEFI Configurator

Средство, используемее для создания пакетов SEMM, — Microsoft Surface UEFI Configurator. Вы можете скачать Microsoft Surface UEFI Configurator со страницы "Инструменты Surface для [ИТ-специалисты"](https://www.microsoft.com/download/details.aspx?id=46703) в Центре загрузки Майкрософт.
Запустите файл установщика Windows (MSI) Microsoft Surface UEFI, чтобы запустить установку средства. После завершения установки найдите microsoft Surface UEFI Configurator в разделе "Все приложения" меню "Пуск".

>[!NOTE]
>Microsoft Surface UEFI Configurator поддерживается только в Windows 10.

##  <a name="create-a-surface-uefi-configuration-package"></a>Создание пакета конфигурации UEFI Surface

Пакет конфигурации UEFI Surface выполняет роль применения новой конфигурации параметров UEFI Surface к устройству Surface, управляемому с помощью SEMM, и роль регистрации устройств Surface в SEMM. Для создания пакета конфигурации необходимо, чтобы сертификат подписи использовался с SEMM для защиты конфигурации параметров UEFI на каждом устройстве Surface. Дополнительные сведения о требованиях к сертификату SEMM см. в [режиме управления Microsoft Surface Enterprise.](https://technet.microsoft.com/itpro/surface/surface-enterprise-management-mode)

Чтобы создать пакет конфигурации UEFI Surface, выполните следующие действия.

1. Откройте Microsoft Surface UEFI Configurator из меню "Пуск".
2. Нажмите **Начать**.
3. Щелкните **"Пакет конфигурации",** как показано на рисунке 1.

   ![Создание пакета для регистрации в SEMM](images/surface-ent-mgmt-fig1-uefi-configurator.png "Create a package for SEMM enrollment")

   *Рисунок 1. Выбор пакета конфигурации для создания пакета для регистрации и настройки SEMM*

4. Щелкните **"Защита** сертификатов", чтобы добавить экспортный файл сертификата с закрытым ключом (PFX), как показано на рисунке 2. Перейдите к расположению файла сертификата, выберите его и нажмите кнопку **"ОК".**

   ![Добавление сертификата SEM и пароля UEFI Surface в пакет конфигурации](images/surface-ent-mgmt-fig2-securepackage.png "Add the SEM certificate and Surface UEFI password to configuration package")

   *Рисунок 2. Добавление сертификата SEMM и пароля UEFI Surface в пакет конфигурации UEFI Surface*

5. Когда вам будет предложено подтвердить пароль сертификата, введите и подтвердите пароль для файла сертификата, а затем нажмите кнопку **"ОК".**
6. Щелкните **"Защита** паролем", чтобы добавить пароль в Surface UEFI. Этот пароль потребуется при загрузке в UEFI. Если этот пароль не введен, отображаются **** только сведения **** о компьютере, сведения об управлении предприятием и выходе. **** **** Этот шаг является необязательным.
7. При отображении запроса введите и подтвердите выбранный пароль для Surface UEFI, а затем нажмите кнопку **"ОК".** Если необходимо очистить существующий пароль UEFI Surface, оставьте поле пароля пустым.
8. Если вы не хотите, чтобы пакет UEFI Surface применялся к конкретному устройству, на странице "Выбор типа **Surface",** которую вы хотите нацелить, щелкните ползунок под соответствующим изображением Surface Book или Surface Pro 4, чтобы он был в положении "Вы отключено". **** (Как показано на рисунке 3.)
   > [!NOTE] 
   > Необходимо выбрать устройство, так как по умолчанию ни одно устройство не выбрано.

   ![Выбор устройств для совместимости пакетов](images/surface-semm-enroll-fig3.jpg "Choose devices for package compatibility")

   *Рисунок 3. Выбор устройств для совместимости пакетов*

9. Нажмите кнопку **Далее**.
10. Если вы хотите деактивировать компонент на управляемых устройствах Surface, на странице "Выбор компонентов, которые необходимо активировать или отключить", щелкните ползунок рядом с любым устройством или группой устройств, которые вы хотите отключить, чтобы ползунок был в состоянии **отключения.** **** (Показано на рисунке 4.) Конфигурация по умолчанию для каждого устройства находится в **режиме "В сети".** Нажмите **кнопку сброса,** чтобы вернуть все ползунки в положение по умолчанию.

    ![Отключение или отключение компонентов Surface](images/surface-ent-mgmt-fig3-enabledisable.png "Disable or enable Surface components")

    *Рисунок 4. Отключение или отключение отдельных компонентов Surface*

11. Нажмите кнопку **Далее**.
12. Чтобы включить или отключить дополнительные параметры в Surface UEFI или отобразить страницы UEFI Surface, на странице "Выбор дополнительных параметров для **** устройств" щелкните ползунок рядом с нужным параметром, чтобы включить или отключить этот параметр **(как** показано на рисунке 5). **** В разделе "Передняя страница **UEFI"** вы можете **** использовать **** ползунки для **безопасности,** устройств и загрузки, чтобы управлять страницами, доступными пользователям, которые загружаются на UEFI Surface. (Дополнительные сведения о параметрах UEFI Surface см. в под [управлением параметров UEFI Surface.)](https://technet.microsoft.com/itpro/surface/manage-surface-uefi-settings) Нажмите **кнопку** "Сборка", когда закончите выбирать параметры для создания и сохранения пакета.

    ![Управление расширенными настройками UEFI Surface и страницами UEFI Surface](images/surface-ent-mgmt-fig4-advancedsettings.png "Control advanced Surface UEFI settings and Surface UEFI pages")

    *Рисунок 5. Управление расширенными настройками UEFI Surface и страницами UEFI Surface с помощью SEMM*

13. В **диалоговом** окне "Сохранить как" укажите имя пакета конфигурации UEFI Surface, перейдите к расположению, в котором вы хотите сохранить файл, а затем нажмите кнопку **"Сохранить".**
14. После создания и с сохранения пакета отображается **страница "Успешно".**

>[!NOTE]
>Зафиксировать отпечаток сертификата, отображаемой на этой странице, как показано на рисунке 6. Эти символы необходимы для подтверждения регистрации новых устройств Surface в SEMM. Нажмите **кнопку** "Конец", чтобы завершить создание пакета и закроем Microsoft Surface UEFI Configurator.

![Отображение символов отпечатка сертификата](images/surface-ent-mgmt-fig5-success.png "Display of certificate thumbprint characters")

*Рисунок 6. Последние два символа отпечатка сертификата отображаются на странице "Успешно"*

Теперь, когда вы создали пакет конфигурации UEFI Surface, вы можете зарегистрировать или настроить устройства Surface.

>[!NOTE]
>При создании пакета конфигурации UEFI Surface на рабочем столе создается файл журнала с подробными сведениями о параметрах и параметрах пакета конфигурации.

##  <a name="enroll-a-surface-device-in-semm"></a>Регистрация устройства Surface в SEMM
При выполнении пакета конфигурации UEFI Surface сертификат SEMM и файлы конфигурации UEFI Surface находятся в хранилище микропрограмм устройства Surface. При перезагрузке устройства Surface UEFI обрабатывает эти файлы и начинает процесс применения конфигурации UEFI Surface или регистрации устройства Surface в SEMM, как показано на рисунке 7.

![Процесс SEMM для настройки UEFI Или регистрации Surface](images/surface-semm-enroll-fig7.png "SEMM process for configuration of Surface UEFI or enrollment")

*Рисунок 7. Процесс SEMM для настройки UEFI Surface или регистрации устройства Surface*

Перед началом процесса регистрации устройства Surface в SEMM убедитесь, что у вас есть последние два символа отпечатка сертификата. Эти символы необходимы для подтверждения регистрации устройства (см. рис. 6).

Чтобы зарегистрировать устройство Surface в SEMM с помощью пакета конфигурации UEFI Surface, выполните следующие действия.

1. Запустите MSI-файл пакета конфигурации UEFI Surface на устройстве Surface, которое вы хотите зарегистрировать в SEMM. Это подтвердит файл конфигурации UEFI Surface во микропрограмме устройства.
2. Установите для **условия** лицензионного соглашения условия лицензионного соглашения и нажмите кнопку **"Установить",** чтобы начать процесс установки.
3. Нажмите **кнопку** "Готово", чтобы завершить установку пакета конфигурации UEFI Surface, и перезапустите устройство Surface, когда вам будет предложено это сделать.
4. Surface UEFI загрузит файл конфигурации и определит, что SEMM не включен на устройстве. Затем Surface UEFI начнет процесс регистрации SEMM следующим образом:
   * Surface UEFI проверит, содержит ли файл конфигурации SEMM сертификат SEMM.
   * Surface UEFI будет предложено ввести последние два символа отпечатка сертификата, чтобы подтвердить регистрацию устройства Surface в SEMM, как показано на рисунке 8.

      ![Для регистрации SEMM требуются последние два символа отпечатка сертификата](images/surface-semm-enroll-fig8.png "SEMM enrollment requires last two characters of certificate thumbprint")

      *Рисунок 8. Для регистрации в SEMM требуются два последних символа отпечатка сертификата*

   * Surface UEFI сохраняет сертификат SEMM во микропрограммах и применяет параметры конфигурации, указанные в файле конфигурации UEFI Surface.
   
5. Устройство Surface теперь зарегистрировать в SEMM и будет загрузиться в Windows.

Вы можете убедиться, что устройство Surface успешно зарегистрировать в SEMM, найдите пакет конфигурации **Microsoft Surface** в программах и компонентов **(как** показано на **** рисунке 9) или события, хранимые в журнале **Настройщика Microsoft Surface UEFI,** найденных в журнале приложений и служб в окне просмотра событий (как показано на рисунке 10).

##  <a name="configure-surface-uefi-settings-with-semm"></a>Настройка параметров UEFI Surface с помощью SEMM

После регистрации устройства в SEMM можно запустить пакеты конфигурации UEFI Surface, подписанные с помощью того же сертификата SEMM, чтобы применить новые параметры UEFI Surface. Эти параметры применяются автоматически при следующей загрузке устройства без участия пользователя. Вы можете использовать такие решения развертывания приложений, как Microsoft Endpoint Configuration Manager, для развертывания пакетов конфигурации UEFI Surface на устройствах Surface для изменения параметров в Surface UEFI или управления ими.

Дополнительные сведения о развертывании файлов установщика Windows (MSI) с помощью Configuration Manager см. в сведениях о развертывании приложений и управлении ими с помощью [Microsoft Endpoint Configuration Manager.](https://technet.microsoft.com/library/mt627959)

Если вы зафиксировали пароль для Surface UEFI, пользователи без пароля, которые попытаются **** загрузиться **** на UEFI Surface, будут иметь только сведения о компьютере, **** управление предприятием и страницы выхода. ****

Если вы не заблокировали Surface UEFI паролем или пользователь введет пароль правильно, параметры, настроенные с помощью SEMM, будут затемнены (недоступны), а текст, которым управляет ваша организация, будет отображаться в верхней части страницы, как показано на рисунке 12.

![Параметры, управляемые SEMM, отключены в Surface UEFI](images/surface-semm-enroll-fig12.png "Settings managed by SEMM disabled in Surface UEFI")

*Рисунок 12. Параметры, управляемые SEMM, будут отключены в Surface UEFI*
